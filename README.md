# 电商用户行为分析

本项目基于 Kaggle 公开的电商用户行为数据集，利用 Python（Pandas, Matplotlib, Seaborn）进行数据清洗、用户行为漏斗分析及设备端对比，旨在识别转化关键节点，为提升购买转化率提供数据驱动的优化建议。

## 数据集

- **来源**：[Kaggle - E-Commerce User Behavior and Purchase Propensity](https://www.kaggle.com/datasets/snehasishhaldar/e-commerce-user-behavior-and-purchase-propensity)
- **规模**：455,401 条记录，25 个字段
- **关键字段**：
  - `UserID`：用户唯一标识
  - `basket_icon_click`：是否点击购物车图标
  - `saw_checkout`：是否查看结算页
  - `sign_in`：是否完成登录
  - `ordered`：是否最终购买
  - `device_mobile` / `device_computer`：设备类型标识
- **数据质量**：经检测，无缺失值，可直接用于分析。

## 分析内容

### 1. 用户转化漏斗
按用户 ID 去重，构建从“点击购物车”到“完成购买”的四步转化漏斗：

| 阶段 | 用户数（去重） | 转化率（相对上一步） |
|------|----------------|----------------------|
| 点击购物车 | 45,153 | 100.00% |
| 查看结算页 | 20,564 | 45.54% |
| 登录 | 20,505 | 99.71% |
| 购买 | 13,534 | 66.00% |

- **最大流失节点**：从“点击购物车”到“查看结算页”，流失率超过 54%。
- **登录环节高效**：进入结算页的用户中 99.71% 完成登录，说明登录不是主要障碍。
- **支付环节仍有空间**：登录后仍有 34% 用户未购买，可能受支付方式、运费、价格等因素影响。

### 2. 设备端行为对比
对比移动端与电脑端在各环节的转化率：

| 指标 | 移动端 | 电脑端 |
|------|--------|--------|
| 总用户数 | 309,994 | 88,448 |
| 点击购物车率 | 10.25% | 9.81% |
| 查看结算页率 | 7.32% | 10.68% |
| 登录率 | 8.24% | 11.50% |
| **购买率** | **3.60%** | **6.20%** |

- **电脑端购买率（6.2%）远高于移动端（3.6%）**。
- 移动端在“查看结算页”和“登录”环节转化率明显低于电脑端，是移动端转化低的主要原因。

## 业务建议

基于分析结果，提出以下优化方向：

1. **优化购物车→结算页引导**  
   - 增加悬浮“去结算”按钮，展示预计送达时间或库存紧张提示，提高用户进入结算页的意愿。
   - 支持购物车页面直接预览部分收货信息，减少心理负担。

2. **专项提升移动端体验**  
   - 推广手机号一键登录或微信/支付宝授权登录，简化移动端登录流程。
   - 优化移动端结算页加载速度，针对手机屏幕优化地址填写表单（智能识别、减少输入）。

3. **精细化运营策略**  
   - 向移动端加购未购用户推送限时优惠券、包邮券。
   - 支付页面突出分期免息等选项，降低决策压力。
   - 对“已登录未购买”用户，24小时内推送提醒，召回潜在订单。

## 结论

通过本次分析，定位了电商转化中的核心瓶颈（购物车→结算页流失过半）和关键短板（移动端体验）。优化结算页引导和移动端登录体验，有望显著提升整体转化率。后续可引入机器学习模型进一步预测用户购买概率，实现精准营销。
